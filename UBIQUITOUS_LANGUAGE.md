# Calen 股票领域统一语言

| 术语                        | 统一含义                                                                                                         | 避免使用                               |
| --------------------------- | ---------------------------------------------------------------------------------------------------------------- | -------------------------------------- |
| 标的（Instrument）          | 可被查询的股票、ETF、指数等金融资产                                                                              | 用“股票”泛指所有资产                   |
| `InstrumentRef`             | 跨 Provider 的稳定标的引用，含市场、交易所、资产类型、币种和展示名                                               | Provider 私有代码作为全局 ID           |
| 市场（Market）              | CN、HK、US 等交易地域与规则集合                                                                                  | 与交易所混用                           |
| 交易所（Exchange）          | SSE、SZSE、HKEX、NASDAQ 等具体挂牌场所                                                                           | 与市场混用                             |
| 能力（Capability）          | Provider 可提供的数据或分析类别                                                                                  | 把“工具名”当领域能力                   |
| Provider                    | 外部或本地的数据提供适配器                                                                                       | 数据源、接口、网站混作实现对象         |
| 证据结果（Evidence Result） | 带来源、时效、缓存和警告的结构化结果                                                                             | 无出处的模型结论                       |
| 数据截至时间（`asOf`）      | 数据描述的业务时间；Provider 无法证明业务时间时使用“未知”而不是 `retrievedAt`                                    | 与获取时间混用、用当前时间冒充资料时效 |
| 获取时间（`retrievedAt`）   | Calen 实际取得数据的时间                                                                                         | 与行情时间混用                         |
| 聚合截至时间                | 多条证据合并时采用最旧的可证明 `asOf`；任一关键证据时间未知时不得用较新时间掩盖                                  | 取最新时间让整批结果看起来更新         |
| 部分结果（`partial`）       | 部分能力或来源失败，但仍有可用证据                                                                               | 将缺失字段默认为零                     |
| 不可用（`unavailable`）     | 无可靠结果可返回                                                                                                 | 由 AI 猜测补全                         |
| 市场简报（Market Brief）    | 某时间窗口的证据化市场概览                                                                                       | 自动生成的投资建议                     |
| 研究简报（Research Brief）  | 围绕标的组织事实、论据、风险和待验证项                                                                           | 买卖信号                               |
| 实验分析                    | 指标、评分、Evaluator、策略或回测产生的非事实层结果                                                              | 机构保证、收益预测                     |
| Sidecar                     | 由 Tauri 管理、通过 stdio JSON-RPC 通信的股票子进程                                                              | 独立 Web 服务                          |
| 回退（Fallback）            | 当前 Provider 失败后按能力选择替代 Provider                                                                      | 静默掩盖所有错误                       |
| 熔断（Circuit Breaker）     | Provider 连续失败后暂时停止请求                                                                                  | 永久禁用 Provider                      |
| Provider 健康状态           | 按 Provider、能力和市场记录已观测结果；`unknown` 表示未探测，不代表服务不可用                                    | 将备用源未调用误报为服务降级           |
| 节流（Throttle）            | 请求发出前按 Provider 共享总节拍，能力可追加更细限制；等待可取消并受总超时约束                                   | 让同一上游的不同能力绕过总频率限制     |
| 策略注册表                  | 一组具名、版本化、可复算的实验研究规则                                                                           | 自动交易策略商店                       |
| 信号融合                    | 汇总多个实验策略的方向、权重和原因，输出研究 verdict                                                             | 目标仓位或下单指令                     |
| Gateway 资产投影            | 向远程 Gateway 提供的脱敏占位，不包含本地持仓和流水明文                                                          | 本地资产账本副本                       |
| 研究子页签（Research Tab）  | 研究页内按问题域划分的固定视图：概览、财务、股东与分红、资金与消息、量化实验；每个页签只请求回答该问题所需的能力 | 一次拉取全部能力再整页平铺             |
| AI 研究简报（AI Brief）     | 用户在概览页手动触发、仅基于已取得证据包生成的模型摘要                                                           | 选中标的即自动生成、脱离证据的模型结论 |
| 组合（Portfolio）           | 用户定义的一组持仓和交易流水                                                                                     | 券商账户                               |
| 交易流水（Transaction）     | 用于本地核算的买卖、费用、分红、拆股或调整记录                                                                   | 真实下单                               |

领域不变量：股票能力首版只读；任何结论都不能脱离来源和时间；实验分析必须与事实数据分区；AI 读取组合必须由用户当前请求明确授权。
