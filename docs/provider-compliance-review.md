# 股票数据源上线审查（首版）

- 审查日期：2026-07-15
- 范围：腾讯行情、东方财富
- 官方页面核查日期：2026-07-15
- 结论：工程能力可以继续开发和本地验证，但公开 Release 在取得书面授权/合规批准前必须被阻断。不得把网页可访问性解释为数据复制、传播或再分发许可。

## 访问与产品约束

| Provider | 访问方式                                    | 默认状态                     | 首版约束                                                               |
| -------- | ------------------------------------------- | ---------------------------- | ---------------------------------------------------------------------- |
| 腾讯行情 | 与公开行情页面相同的免登录 HTTP 接口        | 开发启用；Release 受门禁阻断 | 仅快照、日 K 和基础证券身份；无 SLA，失败时回退或降级                  |
| 东方财富 | 公开网页使用的行情、F10、公告和市场专题接口 | 开发启用；Release 受门禁阻断 | 仅按需访问；短 TTL 缓存、超时、熔断和限流；公告 PDF URL 不作可用性保证 |

## 官方条款依据

- 腾讯法律声明：https://www.tencent.com/zh-cn/statement.html
  - 页面声明腾讯发行或共同发行作品受版权保护，超越合理使用且未经书面许可的使用将保留追责权利。
- 腾讯服务协议：https://www.tencent.com/zh-cn/service-agreement.html
- 东方财富法律声明：https://about.eastmoney.com/home/legal
  - “版权”章节禁止未经书面许可复制、修改、转载、传播或散发网站内容。
  - “特别声明”章节明确，未经交易所事先书面同意不得复制、传播、转播、演示或以其他方式散发行情信息。
- 东方财富服务协议：https://about.eastmoney.com/home/protocol

上述条款没有向 Calen 提供公开的数据再分发许可。GitHub Release workflow 因此要求仓库变量 `CALEN_STOCK_PROVIDER_TERMS_APPROVED=true`；该变量只能在项目维护者取得书面授权或完成正式法律/合规批准后设置。

实现不得携带登录 Cookie、绕过访问控制、抓取用户账户数据或将站点响应打包进安装器。网页接口中的固定 `token/ut` 参数属于公开页面请求参数，不是用户密钥；不得把它们描述为官方开放 API 凭据。

所有结果必须显示 Provider、`asOf`、`retrievedAt`、缓存状态和警告。默认缓存为 5 分钟，市场数据只按用户请求或定时报告任务获取，不做秒级后台轮询。原始响应不进入 Gateway，不作为 Calen 数据产品再分发。

## 风险与停止条件

这些网页接口没有 Calen 可依赖的稳定 SLA，站点条款、字段和访问策略可能变化。本记录是工程风险审查，不是法律意见，也不构成发布批准。若出现验证码/登录门禁、收到权利人通知，或 Provider 持续返回 403/429，维护者必须立即通过数据源设置或版本更新禁用对应 Provider。

新浪、BaoStock、ZZShare、Tushare、TickFlow、Fuyao 尚未完成适配器与独立条款审查，因此默认禁用；即使本地已保存 Key，也不得注册为可用 Provider。
