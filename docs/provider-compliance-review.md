## 官方条款依据

- 腾讯法律声明：https://www.tencent.com/zh-cn/statement.html
  - 页面声明腾讯发行或共同发行作品受版权保护，超越合理使用且未经书面许可的使用将保留追责权利。
- 腾讯服务协议：https://www.tencent.com/zh-cn/service-agreement.html
- 东方财富法律声明：https://about.eastmoney.com/home/legal
  - “版权”章节禁止未经书面许可复制、修改、转载、传播或散发网站内容。
  - “特别声明”章节明确，未经交易所事先书面同意不得复制、传播、转播、演示或以其他方式散发行情信息。
- 东方财富服务协议：https://about.eastmoney.com/home/protocol

上述腾讯与东方财富公开条款本身没有向 Calen 提供通用的数据再分发许可。项目维护者已在仓库外完成首版默认数据源的正式合规批准并确认可以发布；本记录仅保留工程侧确认，不替代或公开批准材料。其余六个 Provider 的端点、鉴权和能力来自当前本地实现及上游技术资料，本次只完成工程盘点，没有完成正式服务条款、隐私、缓存、再分发、商业使用或地域限制审查，也没有取得批准。GitHub Release workflow 因此要求仓库变量 `CALEN_STOCK_PROVIDER_TERMS_APPROVED=true`，且该变量仅覆盖首版默认启用的数据源。

实现不得携带登录 Cookie、绕过访问控制、抓取用户账户数据或将站点响应打包进安装器。网页接口中的固定 `token/ut` 参数属于公开页面请求参数，不是用户密钥；不得把它们描述为官方开放 API 凭据。

Tushare、TickFlow 和 Fuyao 只有在用户显式启用并配置必要 Key 后才注册；Key 只注入对应请求并驻留本地 sidecar 内存，不进入 URL、状态、日志或 Gateway。ZZShare 可在用户显式启用后使用服务定义的 `anonymous` 请求头，也可使用可选 Key；BaoStock 使用服务内置匿名 TCP 登录。这些凭据或匿名模式只说明技术访问条件，不代表条款许可、数据所有权或公开再分发权。

所有结果必须显示 Provider、`asOf`、`retrievedAt`、缓存状态和警告。默认缓存为 5 分钟，市场数据只按用户请求或定时报告任务获取，不做秒级后台轮询。原始响应不进入 Gateway，不作为 Calen 数据产品再分发。

## 风险与停止条件

这些数据接口没有 Calen 可依赖的稳定 SLA，服务条款、字段和访问策略可能变化。本记录是工程风险审查，不是法律意见，也不构成发布批准。若出现验证码/登录门禁、收到权利人通知，或 Provider 持续返回 403/429，维护者必须立即通过数据源设置或版本更新禁用对应 Provider。

新浪、BaoStock、Tushare、TickFlow、ZZShare 和 Fuyao 已完成对应范围的本地 mock/契约测试，但均未完成独立条款审批，因此保持默认禁用。不得仅因适配器已实现、Key 已保存、匿名接口可访问或本地测试通过，就自动启用 Provider、设置 `CALEN_STOCK_PROVIDER_TERMS_APPROVED=true`，或宣称可以公开分发其数据。
