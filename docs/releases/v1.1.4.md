# Calen v1.1.4

> 本版本新增设备本地应用代理和可选离线翻译，并改进 Skills 商店、长对话浏览与上下文压缩稳定性。

## 应用网络代理

- 设置中新增“应用网络代理”，支持直连、跟随 Windows 系统代理和手动 HTTP 代理。
- Provider 请求、Skills 商店、股票研究、应用更新和离线模型下载统一遵循所选网络策略。
- 可选将代理环境应用到 Shell 与后台子进程；默认不会改变用户自己的终端环境。
- 手动代理或仅 PAC 的系统配置不可用时返回明确错误，不会静默切回直连。
- 代理设置仅保存在当前设备，不进入 Gateway 设置同步；诊断信息不会展示代理凭据。

## 离线翻译与 Skills 商店

- 设置中新增“翻译”页面，可选择仅远程、本地优先或仅本地模式。
- 可明确下载 Qwen3-0.6B Q8_0，也可导入用户自行取得的 HY-MT 或其他 GGUF 模型；应用不会自动下载模型权重。
- 下载支持断点续传、取消、固定来源版本、大小与 SHA-256 校验，并在校验成功后原子安装。
- 内置模型和导入模型在使用前都会核对完整性，同尺寸但内容被修改的文件不会被加载。
- Skills 商店翻译会显示实际使用的本地或远程来源；本地优先发生远程回退时会展示警告，“仅本地”模式绝不上传待翻译文本。
- Windows 安装包携带从固定 llama.cpp 源码提交自行构建的 CPU x64 运行时，不包含任何离线模型权重。

## 聊天体验

- 改进长对话自动滚动与跟随逻辑，减少生成过程中意外跳动或抢夺阅读位置。
- 新增消息导航栏，便于在较长对话中快速定位用户消息和助手回复。
- 加强压缩摘要的结构校验和回退处理，降低异常摘要破坏后续上下文的概率。

## 隐私与兼容性

- 离线翻译运行时仅绑定随机的本机回环端口，并使用随机 API Key；应用退出、重启和更新安装前会清理进程树。
- 模型路径、翻译原文、Provider Key 和代理凭据不会写入普通诊断信息或 Gateway 同步数据。
- 离线翻译仍属于实验性功能；用户导入模型时需自行确认模型来源、许可证和适用地区。
- 本版本继续只发布 Windows x64 的 `Setup.exe`、`.msi`、对应 updater 签名和 Windows-only `latest.json`。
