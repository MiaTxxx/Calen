# Calen v1.1.7

> This release adds flexible scheduled-task intervals, a Windows shell preference for Agent tools, and resilient reconnects for transient model/API stream failures.

## Scheduled tasks

- Scheduled tasks can now use a fixed interval expressed as every N minutes, hours, or days.
- The editor generates a true fixed-duration `@every` schedule and shows it before saving.
- Existing advanced cron expressions remain available for schedules that do not fit a fixed interval.
- Interval values are validated as whole numbers from 1 through 999.

## Windows shell preference

- Settings now lets Windows users choose Auto, Bash, or PowerShell for Agent shell tools.
- Bash mode prefers Git Bash or `bash.exe` and falls back to PowerShell when Bash is unavailable.
- PowerShell mode uses the PowerShell command chain without selecting Bash.
- The preference applies to Agent Bash and ManagedProcess tools; interactive terminal sessions keep their own behavior.

## Model/API stream reconnects

- Transient provider and transport failures can restart a model stream before any visible or executable response content is committed.
- Retries use bounded exponential backoff with jitter and stop after five total attempts by default.
- Authentication, permission, billing, and quota failures fail immediately instead of being retried.
- Cancellation interrupts an active retry delay, and committed responses are never restarted automatically.

## Repository presentation

- The README release link is now plain text, removing the external dynamic badge that could render as a broken image when the badge service timed out.
- English and Simplified Chinese READMEs now highlight the current release features and link to these notes.
- Existing compatibility storage paths and internal identifiers remain unchanged.
