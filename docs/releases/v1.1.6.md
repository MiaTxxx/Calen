# Calen v1.1.6

> This release improves conversation continuity, layout density, personalization, provider management, recent history, Windows activity feedback, and the stock-research workspace.

## Drafts and context control

- Unsent rich-text drafts now persist per conversation, including large pastes, working directories, and attachment references.
- Desktop drafts use SQLite, while WebUI drafts remain in browser-local IndexedDB.
- Switching conversations, opening Settings, changing pages, or restarting no longer discards unfinished input.
- A new context reset action keeps visible history but excludes earlier messages and summaries from subsequent model requests.
- Manual reset boundaries persist in history and are relayed through the Gateway conversation event stream.

## Chat layout and appearance

- The transcript and composer now share a persistent content width that can be adjusted by dragging.
- The composer height can be resized independently, with responsive limits for small screens and coarse pointers.
- Settings adds narrow, standard, wide, and full-width presets plus exact layout controls.
- A controlled theme editor covers major surfaces, light and dark palettes, font families, font sizes, JSON import/export, and reset actions.
- The Windows title bar has stronger contrast and clearer top and bottom boundaries.

## Models and providers

- The provider editor now uses a wider two-column layout with endpoint settings on the left and models on the right.
- Model order remains stable when models are enabled, edited, or added; the list no longer jumps to the top.
- Search, added-state filters, counts, row highlighting, new-model markers, and inline token limits improve large catalogs.
- Provider name and URL suggestions now use application-managed history without storing API keys.
- Recent provider entries can be paused, hidden individually, restored, or cleared by provider type.

## Conversations and Windows activity

- Recent conversations are grouped into pinned, today, yesterday, and dated sections.
- AI-generated titles can be disabled or assigned a dedicated model, with locale-aware length limits and local fallback titles.
- Windows displays indeterminate taskbar activity while one or more conversations are running and clears it on every terminal state.
- Taskbar API failures are logged without affecting agent execution.

## Stock research workspace

- The stock hub has been separated into focused market, research, source, laboratory, and experimental analysis modules.
- Recent instruments and shared research surfaces reduce repeated navigation and keep evidence details consistent.
- Existing source, freshness, warning, and uncertainty requirements remain enforced for stock results.

## Compatibility and storage

- Existing `.liveagent` data paths, `LIVEAGENT_*` compatibility variables, application identifiers, and stored history remain supported.
- New local preferences are normalized with safe defaults and are not copied into ordinary Gateway synchronization.
